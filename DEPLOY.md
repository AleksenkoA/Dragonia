# Инструкция по деплою на GitHub Pages

## Быстрый деплой

### Шаг 1: Подготовка файлов

Убедитесь, что все файлы закоммичены:

```bash
git add .
git commit -m "Prepare for GitHub Pages deployment"
git push origin main
```

### Шаг 2: Настройка GitHub Pages

1. Перейдите на GitHub в ваш репозиторий: https://github.com/AleksenkoA/Dragonia
2. Откройте **Settings** (Настройки)
3. В левом меню найдите раздел **Pages**
4. В разделе **Source** выберите:
   - **Branch**: `main`
   - **Folder**: `/ (root)`
5. Нажмите **Save**

### Шаг 3: Ожидание деплоя

GitHub Pages автоматически начнет деплой. Обычно это занимает 1-2 минуты.

После завершения деплоя ваш сайт будет доступен по адресу:
**https://aleksenkoa.github.io/Dragonia/**

> **Примечание:** Если вы хотите использовать кастомный домен, настройте его в разделе **Custom domain** в настройках Pages.

## Автоматический деплой (опционально)

Если вы хотите автоматический деплой при каждом push, можно использовать GitHub Actions. Файл `.github/workflows/deploy.yml` уже создан (если нужно).

## Проверка деплоя

После деплоя проверьте:
- ✅ Все страницы открываются корректно
- ✅ Изображения загружаются
- ✅ CSS и JavaScript работают
- ✅ Адаптивность на мобильных устройствах

## Обновление сайта

После каждого изменения просто делайте:

```bash
git add .
git commit -m "Update content"
git push origin main
```

GitHub Pages автоматически обновит сайт в течение 1-2 минут.
